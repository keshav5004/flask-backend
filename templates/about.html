Detection Engine Mechanics

WebShield utilizes a multi-layered approach to evaluate the safety profile of a website domain. The algorithm is split into two primary phases: Exact Match Lookup and Heuristic Similarity Scanning.

1. Exact Match Threat Database

Every input URL is first parsed and normalized into its base domain. This removes the subdomains (e.g. www.), path endpoints, request queries, and ports to get a standardized hostname (e.g. unsafe-portal.co.uk).

The engine queries an in-memory threat cache loaded directly from the system database. If the normalized domain exists in the blacklist, it is flagged as Malicious (100% Risk Score) instantly.

2. Similarity & Typo Heuristics

If no exact match exists, the system passes the domain into the Similarity Checker Engine, which scans for active typosquatting and impersonation indicators:

  • Fuzzy String Matching: Computes the Levenshtein-like distance ratios against top high-traffic brands and existing blacklisted targets.
  • Homoglyph Detection: Scans for lookalike character mappings (such as Latin o swapped with Cyrillic characters, or substituting rn for m).
  • Subdomain Nesting: High frequency of subdomains (e.g. paypal.com.verify-login.net) triggers brand impersonation alerts.
  • Keyword Flagging: Matches typical phrases used in social engineering links (e.g. secure, signin, alert, recover).

3. Composite Risk Index (CRI)

A cumulative risk score is calculated on a scale of 0 to 100. Based on this value, the domain is classified into one of the following five security ratings:

Safe (0-20)
Low (21-40)
Medium (41-60)
High (61-80)
Malicious (81-100)
